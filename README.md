# remont-calc

PSEO-сайт калькуляторов ремонта. Astro + статика. Деплой на Cloudflare Pages (бесплатно, без ограничений по трафику). Монетизация — РСЯ.

**Состояние на момент сборки:** 33 готовые посадочные страницы под расчёт обоев + универсальный калькулятор. Архитектура рассчитана на добавление новых калькуляторов (ламинат, краска, плитка, штукатурка и т.д.) копированием шаблона.

---

## Локально

```bash
# одноразово: установить bun (если не стоит)
curl -fsSL https://bun.sh/install | bash
export PATH="$HOME/.bun/bin:$PATH"

# работа
bun install
bun run dev      # http://localhost:4321
bun run build    # сборка в ./dist
```

## Деплой: 5 шагов

### 1. Купить домен (~$3-15/год)
Любой регистратор. **Лучше тематический .ru-домен**, Яндекс к нему благосклоннее. Примеры свободного:
- `remontkalkulator.ru`, `kalk-remont.ru`, `oboi-raschet.ru`
Проверить — на reg.ru или nic.ru.

### 2. Создать репозиторий на GitHub
```bash
cd /Users/mac/remont-calc
git init
git add .
git commit -m "initial"
# создать пустой репо на github.com, потом:
git remote add origin git@github.com:USERNAME/remont-calc.git
git push -u origin main
```

### 3. Деплой на Cloudflare Pages (бесплатно)
1. Зарегаться на [dash.cloudflare.com](https://dash.cloudflare.com/) (бесплатно)
2. Workers & Pages → Create → Pages → Connect to Git → выбрать репозиторий
3. Настройки билда:
   - **Build command:** `bun run build`
   - **Build output directory:** `dist`
   - **Environment variable:** `SITE_URL` = `https://твой-домен.ru`
4. Привязать домен: Pages → твой проект → Custom domains → твой домен. CF сам выдаст NS-записи которые надо вписать у регистратора (или CNAME если NS не хочешь менять).

После этого каждый `git push` будет автоматически переразвёртывать сайт. **Никакой ручной работы.**

### 4. Яндекс.Вебмастер
1. [webmaster.yandex.ru](https://webmaster.yandex.ru/) → Добавить сайт → твой домен.
2. Подтвердить права: способ "HTML-файл". Яндекс даст файл `yandex_XXXX.html` — положить его в `public/` корень, закоммитить, запушить. После деплоя нажать "Проверить".
3. В Вебмастере: **Индексирование → Файлы Sitemap** → добавить `https://твой-домен.ru/sitemap.xml` (уже работает).
4. **Переобход страниц** — добавить главную и `/raschet-oboev/`, отправить на переобход вручную для ускорения первой индексации.

### 5. Подключить монетизацию (РСЯ)
**Важно:** РСЯ принимают сайты с минимальным "возрастом" и контентом. Возможно потребуется подождать 1-2 недели после индексации.

1. [partner2.yandex.ru](https://partner2.yandex.ru/) → подать заявку на сайт.
2. Дождаться модерации (1-7 дней).
3. После одобрения: Создать рекламные блоки → получить **3 blockId** (для top, middle, bottom).
4. Открыть `src/config.ts`, заполнить:
   ```ts
   rsya: {
     enabled: true,
     blockIdTop: 'R-A-XXXXXX-1',
     blockIdMiddle: 'R-A-XXXXXX-2',
     blockIdBottom: 'R-A-XXXXXX-3'
   }
   ```
5. Закоммитить, запушить — CF сам передеплоит, объявления появятся.

### Бонус: Яндекс.Метрика
Создать счётчик на [metrika.yandex.ru](https://metrika.yandex.ru/), скопировать ID, вставить в `src/config.ts` → `metrikaId`. Закоммитить, запушить.

## Архитектура (для масштабирования)

```
src/
├── config.ts                          ← все ID и настройки сайта
├── layouts/Base.astro                 ← общий шаблон (header, footer, meta, метрика, РСЯ скрипт)
├── components/
│   ├── WallpaperCalc.astro            ← виджет калькулятора (vanilla JS)
│   ├── Rsya.astro                     ← слот РСЯ
│   ├── Metrika.astro                  ← код Яндекс.Метрики
│   └── RelatedScenarios.astro         ← внутренняя перелинковка
├── data/wallpaper-scenarios.ts        ← ВСЕ сценарии для обоев = ВСЕ страницы
└── pages/
    ├── index.astro                    ← главная
    ├── sitemap.xml.ts                 ← автогенерируется из scenarios[]
    └── raschet-oboev/
        ├── index.astro                ← раздел "расчёт обоев"
        └── [slug].astro               ← один URL = один сценарий из data
```

### Как добавить новый сценарий под обои
Открыть `src/data/wallpaper-scenarios.ts`, добавить объект в любой массив:

```ts
{
  slug: 'na-komnatu-50-kv-m',
  category: 'area',
  h1: 'Расчёт обоев на комнату 50 м²',
  title: 'Сколько обоев нужно на комнату 50 кв.м — калькулятор',
  description: '...',
  prefill: { length: 8.5, width: 6, height: 2.7, doors: 1, windows: 2 },
  introBlocks: ['...', '...', '...']
}
```

После `bun run build` появится новая страница `/raschet-oboev/na-komnatu-50-kv-m/`, попадёт в sitemap, перелинкуется автоматически. Закоммитил → запушил → задеплоилось.

### Как добавить новый калькулятор (ламинат, краска и т.д.)
1. Скопировать `src/components/WallpaperCalc.astro` → `LaminateCalc.astro`. Поменять формулу.
2. Скопировать `src/data/wallpaper-scenarios.ts` → `laminate-scenarios.ts`. Свои сценарии.
3. Скопировать папку `src/pages/raschet-oboev/` → `src/pages/raschet-laminata/`. Заменить импорты.
4. Добавить раздел в `src/pages/sitemap.xml.ts` (импортировать laminate-scenarios).
5. Добавить ссылку в шапку (`src/layouts/Base.astro`) и на главной.

**Цель:** в идеале 4-5 калькуляторов, ~150-300 страниц всего. Это уже серьёзная PSEO-сетка.

## Что важно знать

### Контент
Все intro-блоки в `wallpaper-scenarios.ts` — **уникальный текст**, не сгенерированный шаблонной подстановкой. Это критично: Яндекс хорошо умеет находить генерированный контент и понижать его. Если будешь добавлять новые сценарии — пиши осмысленные абзацы, не "Калькулятор для {X}. Введите {Y}. Получите {Z}".

### Скорость
Сайт полностью статичный. На CF Pages это значит TTFB <50ms из любой точки. Lighthouse даёт 95+ из коробки.

### Если что-то ломается
- Билд падает на сборке → проверь `bun run build` локально перед пушем
- РСЯ не показывается → проверь `enabled: true` и что блок-ID совпадают
- Метрика не считает визиты → ID без пробелов в config.ts
- Сайтмап пустой → проверь что роуты в `src/pages/sitemap.xml.ts` правильные

## Реалистичные ожидания

- **0-3 мес:** индексация (Яндекс медленный, 1-3 мес до полного захода)
- **3-6 мес:** первые позиции по низкочастотке, 50-500 визитов/день
- **6-12 мес:** если попали в нишу — 2k-10k визитов/день
- **РСЯ RPM в строительной нише:** 60-200 руб за 1000 показов, в среднем ~100
- **При 2k визитов/день и 1.5 показа РСЯ на визит:** ~300 руб/день = ~$3.5/день
- **При 10k визитов/день:** ~1500 руб/день = ~$17/день

То есть **$10-20/день — это таргет на 6-12 мес**. Если ниша не выстрелит — пишешь в опыт и стартуешь следующую (стек одинаковый).
